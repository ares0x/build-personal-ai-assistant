# 00. 前言

## 这个教程是做什么的？

这是一个**从零开始构建个人 AI 助理 Harness** 的实战教程。

我们最终要做的不是又一个简单的 Chatbot，也不是单纯的 Coding Agent，而是一个：

- 越用越懂你的个人助理
- 支持写作、作图、知识管理等多种能力
- 同时具备 CLI、桌面端、REST API、Telegram / 微信 / Slack 等多渠道接入能力
- 底层拥有清晰、可扩展、可维护的 Agent 架构

项目名称叫 **hachimi**。

本教程会完整记录我从零设计、实现、重构到逐步完善这个项目的全过程。

## 为什么要自己做？

2025–2026 年，市面上已经有很多优秀的 AI 编程助手和 Agent 产品，例如 Claude Code、Cursor、Grok Build、Pi 等。

但它们大多存在以下问题：

1. **定位局限**：强绑定在「写代码」场景，很难扩展成真正的个人助理。
2. **不可控**：闭源或核心逻辑不透明，想深度定制非常困难。
3. **记忆能力弱**：大多数产品的长期记忆做得比较浅，很难做到「越来越懂我」。
4. **渠道割裂**：CLI、网页、IM 机器人往往是割裂的。

因此我决定自己动手，做一个**以「个人助理」为核心目标**的 Agent Harness，并把整个过程公开记录下来。

## 我们会参考哪些优秀项目？

本项目不会从零闭门造车，而是站在已有优秀实现的肩膀上：

| 项目 | 我们主要借鉴的点 |
|------|------------------|
| **Pi** | 极简的核心循环、Lazy Skills 设计、干净的代码组织 |
| **Claude Code**（通过公开分析） | 分层 Memory、权限系统、Session 管理、Context 压缩 |
| **Grok Build** | 清晰的模块划分、Headless 优先、插件与 Hooks 思想 |

我们会**取长补短**，而不是简单复制任何一个项目。

## 教程的写作方式

本教程采用「**边开发边记录**」的方式：

- 每个阶段都对应真实可运行的代码
- 不仅讲「怎么做」，更会讲「为什么这么做」
- 会如实记录走弯路和重构的过程
- 尽量保持代码与文档同步

你可以把它当作：

- 一份构建 Agent Harness 的实战手册
- 一次完整的开源项目开发记录
- 一个可以自己动手跟做的学习项目

## 适合谁看？

- 想深入理解 Agent Harness 底层原理的开发者
- 想自己做一个长期个人助理的人
- 对 TypeScript / Node.js 比较熟悉，有一定工程经验的朋友
- 希望学习如何把 Claude Code、Pi、Grok Build 等项目的优点融合起来的人

**不适合**：完全没有编程基础，或者只想要一个开箱即用产品的用户。

## 技术栈与前置要求

- **语言**：TypeScript
- **运行时**：Node.js ≥ 20（推荐使用 pnpm）
- **基础要求**：
  - 熟悉 TypeScript 基础语法
  - 了解异步编程（async/await）
  - 有一定的项目结构意识

我会在教程中尽量把关键设计讲清楚，但不会从零讲解 TypeScript 本身。

## 教程目录预览

1. 项目定位与整体架构设计
2. 基础脚手架与 Monorepo 搭建
3. Phase 1：最小可用核心（Agent Loop + Tool + MockLLM）
4. Phase 2：分层 Memory 设计与实现
5. 交互式 CLI 与基础体验
6. 代码重构实践
7. 接入真实 LLM
8. Skills 系统
9. 持久化与 Session 管理
10. 多渠道支持（REST API、Telegram 等）
11. 权限、安全与可扩展性
12. 总结与后续演进

## 如何跟随本教程？

1. 克隆本教程仓库
2. 同时关注主项目 [hachimi](https://github.com/ares0x/hachimi)（代码实现）
3. 按照章节顺序阅读，并在本地同步实现
4. 每个阶段结束后，建议自己停下来思考：「如果是我会怎么设计？」
