# 00. 前言

## 这个教程是做什么的？

我正在开发一款属于自己的个人 AI 助理，项目名叫 **hachimi**。

在开发过程中，我逐渐意识到：一个 AI 助理的智能化程度，除了需要一颗聪明的大脑（模型本身），还要有一个精心设计的**Harness** ——也就是围绕模型的那一层架构、记忆、工具、权限与扩展机制。

于是我把构建这个 Harness 的过程记录了下来，形成了这份教程。

**本教程的核心目标是**：

> 讲清楚一个可扩展的个人 AI 助理 Harness，应该如何设计与实现。

它重点关注架构思想和关键模块的实现路径，而不是直接交付一个生产级可用的完整产品。

本教程配套的代码在 [hachimi](https://github.com/ares0x/hachimi/tree/tutorial)，本分支会永久保留。

## 为什么要自己做 Harness？

目前市面上已经有很多优秀的 AI 产品，例如 Claude Code、Cursor、Grok Build、Pi 等。它们在各自的领域表现出色，但对我来说仍有一些不满足的地方：

- 多数强绑定在「编程」场景，难以扩展成真正的个人助理
- 长期记忆能力有限，很难做到「越来越懂我」
- 核心逻辑不透明，深度定制成本高
- 多渠道支持（CLI、桌面、IM、API）往往割裂

因此我决定自己动手，从底层 Harness 开始构建。

## 我们会参考哪些项目？

本教程不会从零闭门造车，而是站在已有优秀实现的肩膀上，进行取长补短：

| 项目 | 主要借鉴点 |
|------|------------|
| **Pi** | 极简核心循环、Lazy Skills 设计、干净的代码组织 |
| **Claude Code**（通过公开分析） | 分层 Memory、权限系统、Session 管理、Context 处理 |
| **Grok Build** | 清晰的模块划分、Headless 优先、插件与 Hooks 思想 |

我们会分析它们的设计取舍，并融合成更适合「个人助理」场景的架构。

## 教程的写作方式

本教程采用「边开发边记录」的方式：

- 每个阶段都对应真实可运行的代码
- 不仅讲「怎么做」，更会讲「为什么这么做」
- 会如实记录走弯路和重构的过程
- 重点放在设计思想与架构演进，而非追求功能堆砌

你可以把它当作：

- 一份 Agent Harness 的设计与实现参考
- 一次真实的个人项目开发记录
- 一个可以跟着动手的学习项目

## 适合谁看？

- 想深入理解 Agent Harness 底层原理的开发者
- 想自己动手实现一个可扩展个人助理基础的人
- 对 TypeScript 比较熟悉，有一定工程经验的朋友
- 希望学习如何融合多个优秀开源项目设计的人

**不适合**：只想要一个开箱即用产品，或完全没有编程基础的朋友。

## 技术栈与前置要求

- **语言**：TypeScript
- **运行时**：Node.js ≥ 20
- **包管理**：pnpm
- **基础要求**：熟悉 TypeScript 基础，了解异步编程，有一定的项目结构意识
